OVERVIEW

The Jenga Framework is a PHP Framework which uses a modified form
of MVC (Model - View - Controller) known as Encapsulated MVC (E-MVC)
or Hierachical MVC.
With this approach instead of the entire framework imposing an MVC
structure to be used throughout i.e. all models in a certain folder, all
controllers in another etc. Jenga wraps all the individual MVC in
containers called Elements and in turn communicate with each other.

Comparison between typical MVC and EMVC
The MVC modules are housed in elements grouped according to their
respective functions i.e. customer's element, product's element etc.

DIRECTORY STRUCTURE

Jenga takes the incoming request and routes it to the appropriate element
which in turn makes internal requests to all elements which contain any
relevant information which it needs.

Example: if from the customers element, the client also wanted to view
all the products linked to a particular customer, the customer element
would send a request to the products element for the relevant information.
